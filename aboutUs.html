Tools We Used

Leaflet: This API allowed us to display our two maps easily on our webpage. With this API we were able to add markers to various coordinates on the map where particle data had been taken, add popups containing the average values of the data taken at these points, and add a heatmap overlay corresopnding to the amount of a certain particle found in the air at that point.

Open AQ Platform: This API allowed for us to easily request the air quality measurements for the coordinates that were visible on our maps. The response was a json file, which made it easy to store the data we found into objects and made the data easier to manipulate.

Nominatim API: We used this API to retrieve city data, as well as latitude and longitude coordinates. This helped us implement the city look up.

jQuery: This javascript library helped us simplify our code and made a lot of our functions in javascript easier to understand and work with, such as AJAX. This function allowed for us to get and exchange data on our webpage, without having to reload the whole thing each time we got a new piece of data.

Vue: This framework made it easy for us to organize and maintain all of the data and moving parts of our webpage. We were able to create different objects in the Vue, which helped us structure our data in a way that made it easy to retrieve and manipulate.

  3. There tends to be more data (markers) in more populated cities.
  4. Every particle value was significantly larger in the big cities in China.
